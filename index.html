Artifacts: Raspberry Pi Thermostat Project

  • Original Thermostat Project (CS 350)
    The initial version created for CS 350 demonstrated basic functionality: reading temperature data, controlling LEDs, and displaying output on a 16x2 LCD screen. It provided the foundation for later enhancements and was used to test circuit wiring and device communication.
  • Version 2 – Software Design & Engineering
    This version focused on code structure, maintainability, and system reliability. It added organized error handling, grouped configuration variables, and safe resource management to prevent I/O conflicts and runtime failures. These changes aligned with the software design and engineering category, improving the overall quality and readability of the program.
  • Version 3 – Algorithms & Data Structures
    This milestone introduced algorithmic enhancements that improved performance and stability. A moving average filter was added to smooth sensor readings, a hysteresis function reduced LED flicker, and a scheduling algorithm allowed automatic setpoint changes based on time of day. These changes demonstrated effective use of algorithmic principles to optimize real-world behavior.
  • Version 4 – Database Integration
    The final enhancement implemented SQLite database functionality for persistent data storage. This allowed the thermostat to log temperature readings and user events safely and efficiently. The database structure included tables for readings and events, showing how local data management can improve monitoring and long-term analysis of system performance.

Narratives and Reflections

  • Milestone Two – Software Design and Engineering Narrative
    This reflection discusses the process of improving the original code’s structure and reliability. It focuses on secure coding practices, maintainability, and error management, explaining how Version 2 met the learning outcomes related to design and engineering.
  • Milestone Three – Algorithms and Data Structures Narrative
    This reflection describes the integration of algorithmic logic, such as moving averages, hysteresis, and scheduling. It explores how these updates improved performance and introduced efficient decision-making processes in a real-time embedded system.
  • Milestone Four – Database Integration Narrative
    This reflection explains the addition of the SQLite database and how data persistence expanded the system’s capabilities. It emphasizes the importance of structured data handling, safe commit procedures, and long-term usability in embedded systems.
